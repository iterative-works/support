## Code Style Review

### Critical Issues

#### Missing PURPOSE Comments in `build.mill`
**Location:** `build.mill:1`
**Problem:** The file does not start with the required two `// PURPOSE: ` comment lines. The file begins directly with the Mill bootstrap header (`//| mvnDeps:`).
**Impact:** Violates the CLAUDE.md convention that every code file must start with two `// PURPOSE: ` lines.
**Recommendation:** Add two PURPOSE lines before the Mill bootstrap directives (Mill ignores `//` comment lines that don't start with `//|`).

### Warnings

#### Docstring on `BaseModuleCommon` Understates Its Role
**Location:** `build.mill` (around line 88)
**Problem:** The new trait's doc reads "Shared compile concerns for all IW Support modules — no publish, no scalafix". The negative framing describes what the trait *lacks* rather than what it *is*.
**Recommendation:** Reframe positively: `/** Shared Scala compile configuration inherited by all IW Support modules. */`

#### `NoPublishCrossModule` / `BaseModuleNoPublish` Names Encode Absence
**Location:** `build.mill` (around line 197 and 118)
**Problem:** Names express what the trait *doesn't do* rather than its purpose. The meaningful distinction is that these modules are *internal*, not that they lack publishing.
**Recommendation:** Consider names like `InternalCrossModule`, `BaseInternalModule`, `BaseScalaJSInternalModule`.

#### Redundant Import Comments at Top of `build.mill`
**Location:** `build.mill:11-15`
**Problem:** `// Import IW Mill Support library` and `// Import Scalafix plugin` restate exactly what the following import lines already say.
**Recommendation:** Remove both comments. (Pre-existing; not introduced by this change.)

## Security Review

### Critical Issues

### Warnings

#### `COURSIER_CREDENTIALS` exposed to the "Run tests" step
**Location:** `.github/workflows/publish.yml:46-48`
**Problem:** The `COURSIER_CREDENTIALS` env var is injected into the `Run tests` step, available to any test code via `System.getenv()`. Low-risk with GITHUB_TOKEN, but a least-privilege violation.
**Recommendation:** Resolve dependencies as a distinct pre-test step, then run tests without credentials. Alternatively, document explicitly why tests need registry access. (Note: this mirrors the existing `ci.yml:181` pattern — changing it would deviate from spec.)

#### `COURSIER_CREDENTIALS` embeds actor name as literal in credential string
**Location:** `.github/workflows/publish.yml:47, 56, 66`
**Problem:** Standard Coursier format puts actor username in plain text as part of the credential string.
**Recommendation:** Acceptable as-is; standard pattern.

### Suggestions

#### Consider pinning actions to commit SHA
**Location:** `.github/workflows/publish.yml:21, 24, 29, 37`
GitHub-owned actions pinned to `@v4` is industry-accepted norm. No change required unless project moves to strictest hardening posture.

#### Tag pattern `v*` allows any pre-release tag to trigger Nexus publish
**Location:** `.github/workflows/publish.yml:59-67`
Any `v*` tag triggers EBS Nexus publish. Consider tightening to semver-only pattern (e.g., `'v[0-9]+.[0-9]+.[0-9]+'`) to reduce blast radius of accidental tag pushes. (Note: spec explicitly says `tags: ['v*']`; tightening would deviate from spec — consider as Phase 2 follow-up.)
